# /继续

恢复 `.claude/runs/active.json` 指向的 run，从中断点接着跑。

---

## 执行步骤

### Step 1：检查 active.json

```bash
cat .claude/runs/active.json
```

- **不存在** → 汇报：「没有活跃 run，是否要 `/新需求 <描述>` 起新流水线？」**结束**
- **存在但 status=completed** → 汇报：「上次 run 已完成（终报告位于 `.claude/runs/<run-id>/timeline.log` 末尾），是否要 `/新需求` 起新的？」**结束**
- **存在且 status=running 或 stopped** → 进 Step 2

### Step 2：加载状态

读：
- `.claude/runs/<run-id>/manifest.json`（里程碑列表）
- `.claude/runs/<run-id>/milestones.md`（人类可读进度）
- `.claude/runs/<run-id>/tasks_queue.jsonl`（任务队列）
- `.claude/runs/<run-id>/tasks_done.jsonl`（已完成）
- `.claude/runs/<run-id>/tasks_skipped.jsonl`（跳过）
- `.claude/runs/<run-id>/tasks_running.json`（如果存在 = 上次断在某个任务执行中）
- `.claude/runs/<run-id>/timeline.log` 末尾 20 行（看上次发生了啥）

### Step 3：恢复点判定

按以下顺序找下一步：

#### 情况 A：tasks_running.json 存在且非空
说明上次断在某条任务执行中（可能 dev 跑一半 / 也可能 tester 跑一半 / 也可能 SendMessage 等待中）。

**处理**：
- attempt < 3 → 调用新 developer agent（旧 dev-<ID> agent name 已失效，重新起一个）+ 把 last_test_report 作为初始上下文传过去
- 失败原因都贴出来 + 让新 agent 知道这是延续上轮重试
- 跑完 → tester → 按正常流程（PASS commit / FAIL 累积）

#### 情况 B：tasks_running.json 不存在或空
说明上次干净停在一条任务结束后。

**处理**：
- 在 tasks_queue.jsonl 找下一条 status=pending 且 deps 都 done 的任务
- 进 for-loop 跑下去

#### 情况 C：当前里程碑全部任务完成
**处理**：
- 标 milestones[i].status = done
- push 到远程（commit message: `milestone: M<X> <name> done`）
- 进入下一个 milestone：调 planner 拆任务 → 跑 for-loop

#### 情况 D：所有里程碑完成
**处理**：
- active.json status=completed
- 输出终报告（同 /新需求 Step 5 模板）

### Step 4：恢复时写 timeline.log

```
<ISO8601> RUN_RESUME from_task=<task_id> reason="<user invoked /继续>"
```

### Step 5：进入 for-loop

剩下的逻辑跟 `/新需求` Step 4 完全一致 — 调 developer / tester / cascade skip / 里程碑 push。

---

## 异常处理

| 情况 | 处理 |
|---|---|
| active.json 损坏 / 字段缺失 | 汇报：「active.json 损坏，是否要 `/丢弃 <run-id>` 清理？」结束 |
| run 目录某文件缺失（如 manifest.json） | 同上 |
| tasks_queue.jsonl 行格式错 | 汇报：「<run-id> 的 tasks_queue.jsonl 第 N 行格式错」结束 |
| 用户在两个窗口同时 /继续 | 检测 active.json.last_event_at vs tasks_done.jsonl 末尾时间戳；不一致 → 警告并退出（让用户只保留一个窗口） |
| 里程碑 deps 未满足（前置里程碑 status != done） | 跳到前置里程碑跑；如果前置里程碑也卡（如 M0 用户操作），停下汇报阻塞清单 |

## 汇报模板

恢复成功开始跑时（一次性输出）：

```
## /继续 — 已恢复 run <run-id>

**进度**：M2 里程碑（用户认证），任务 12/20 完成
**上次断点**：P3.6 google_oauth（FAIL 1 轮）
**恢复操作**：重启 dev-P3.6 agent，把上次 FAIL 报告作为初始上下文，进 attempt 2

继续跑直到完成或遇到必停口子 → 终报告。
```

跑完后输出终报告（同 `/新需求` Step 5 模板）。
