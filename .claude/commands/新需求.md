# /新需求 全自动流水线（带里程碑分层 + 跨会话恢复）

用户输入：$ARGUMENTS

**核心原则**：用户只在这一刻说一次需求，**之后全自动跑到底，只在终点汇报**。
所有进度写盘到 `.claude/runs/<run-id>/`，关窗口任意时刻 `/继续` 接着跑。
唯一允许停下的 5 个口子见 CLAUDE.md §7。

按以下 6 步严格执行：

---

## Step 0：起 run + 写状态

1. 检查 `.claude/runs/active.json`：
   - 存在且 `status=running` 或 `stopped` → 提示用户「上次有未完成 run，要 /继续 还是覆盖？」**结束**
   - 否则继续
2. 生成 run-id：`YYYY-MM-DD-HHMM-<5char-slug>`（slug 从 $ARGUMENTS 截）
3. 创建目录 `.claude/runs/<run-id>/`
4. 写 active.json（status=running, started_at=now）
5. 写 timeline.log 第一行：`<ISO> RUN_START title="$ARGUMENTS"`

## Step 1：理解需求 + 红线预检

1. 读 `docs/产品文档.md` 全文
2. 读 `docs/不可自动化清单.md`（识别哪些事 Claude 做不了）
3. 把 `$ARGUMENTS` 解析成结构化需求
4. 对照 `docs/架构与红线.md` 红线 / 不做清单
   - **触红线（必停口子①）** → 调 `redblue` subagent → 输出合稿 → active.json status=stopped + reason=red_line → **结束**

5. 简短汇报（一句话）：「需求理解：X，影响 Y 模块。开始全自动流水线 run-id=Z。」

## Step 2：改产品文档

在 `docs/产品文档.md` 对应章节追加 / 修改 / 删除（用 Edit 工具，**不要 Write 整个文件**）。

**注意**：超大需求（如"开发完整爱奇艺"）通常涉及全项目，可能要改第 4 / 5 / 6 章；按现有结构扩展，不新建顶层章节。

## Step 3：meta-planner 切里程碑

```
Agent({
  description: "切里程碑",
  subagent_type: "meta-planner",
  prompt: "<结构化需求 + run-id + 当前产品文档实现状态>"
})
```

meta-planner 会写 `.claude/runs/<run-id>/manifest.json` + `milestones.md` 并返回汇报。

**判断**：
- 拆出 0 里程碑或冲突（必停口子②）→ 列歧义点 → status=stopped + reason=meta_planner_failed → **结束**
- 拆出里程碑数 ≤ 10（Q3=B）→ 进 Step 4
- 大于 10 → 让 meta-planner 重切（最多 2 次）；2 次还失败 → 必停口子②

**注意**：如果 meta-planner 判断 ≤ 30 任务、≤ 1 角色不需里程碑分层，则跳过本步直接进 Step 4 单层 planner。

## Step 4：里程碑层 for-loop（核心）

读 manifest.json 拓扑序排好里程碑。然后：

```python
for milestone in milestones_topological_order:
    if milestone.status == "done":
        continue

    write_timeline(f"MILESTONE_START {milestone.id}")
    update_active_json(milestone_idx=milestone.idx)

    # 检查是否阻塞用户（必停口子④）
    if milestone.blocks_to_user:
        # 输出阻塞清单
        print_blocking_checklist(milestone)
        active.json.status = "stopped"
        active.json.blocked_reason = "user_action_required"
        write_timeline(f"BLOCKED_USER_INPUT {milestone.id}")
        # 流水线退出，等用户 /继续
        return STOP

    # 调 planner 拆该里程碑任务
    planner_result = Agent(
        description=f"拆解里程碑 {milestone.id}",
        subagent_type="planner",
        prompt=f"<里程碑描述 + run-id>"
    )
    if planner_result.failed:
        # 必停口子②
        active.json.status = "stopped"
        return STOP_AND_REPORT

    # planner 已写 tasks_queue.jsonl 该里程碑的任务
    write_timeline(f"PLANNER_DONE {milestone.id} tasks={N}")

    # 任务层 for-loop（按依赖拓扑序）
    for task in milestone_tasks_topological_order:
        if task.status == "done": continue
        if task.deps_have_skipped:
            # cascade skip Q4=A
            cascade_skip(task)
            continue

        write_running(task, attempt=1)
        write_timeline(f"TASK_START {task.id}")

        attempt = 1
        last_dev_name = None
        while attempt <= 3:
            if attempt == 1:
                dev_result = Agent(
                    subagent_type="developer",
                    name=f"dev-{task.id}",
                    prompt=f"<任务规格 + 验收 + 专属测试规格 + run-id>"
                )
                last_dev_name = f"dev-{task.id}"
            else:
                dev_result = SendMessage(
                    to=last_dev_name,
                    message=f"tester 报告 FAIL（第 {attempt-1} 轮）：\n{tester_report}\n请修复后回复 done。"
                )

            if dev_result.contains("规格歧义"):
                # 必停口子③
                active.json.status = "stopped"
                return STOP_AND_REPORT

            tester_result = Agent(
                subagent_type="tester",
                prompt=f"<任务 ID + dev 改动文件 + 专属测试路径>"
            )

            if tester_result.PASS:
                mark_task_in_role_file(task.id, "[✓]")
                update_progress_total_in_role_file(task.role)
                git_add_and_commit(f"done: {task.id} {task.short_desc}\n\nRun: {run_id}\nMilestone: {milestone.id}")
                tasks_done.jsonl.append({...})
                clear_running()
                write_timeline(f"TASK_PASS {task.id} commit={sha7}")
                break  # 进下一条任务
            else:
                tester_report = tester_result.report
                write_timeline(f"TASK_FAIL {task.id} attempt={attempt}")
                attempt += 1

        if attempt > 3:
            # 必停口子⑤（不停继续）
            mark_task_in_role_file(task.id, "[~]")
            tasks_skipped.jsonl.append({...})
            cascade_skip(task)  # 预跳过下游
            clear_running()
            write_timeline(f"TASK_SKIP {task.id} reason=failed_3_attempts")

    # 里程碑结束
    milestones.md.mark_done(milestone.id)
    manifest.json.milestones[i].status = "done"
    git_push("origin", "main")  # Q5=A 不开 PR；Q2=A 每里程碑 push 一次
    write_timeline(f"MILESTONE_DONE {milestone.id}")
    write_timeline(f"MILESTONE_PUSH origin/main")

# 全部里程碑完成
active.json.status = "completed"
write_timeline(f"RUN_COMPLETE")
```

## 必停口子总览

| ID | 触发 | 处理 |
|---|---|---|
| ① | 触红线 | 调 redblue → status=stopped → 等用户 |
| ② | meta-planner / planner 拆解失败 | 列歧义点 → status=stopped → 等用户 |
| ③ | dev 发现规格歧义 | 列问题 → status=stopped → 等用户改产品文档 + /继续 |
| ④ | blocks_to_user=true 里程碑 | 输出阻塞清单 → status=stopped → 等用户人工完成 + /继续 |
| ⑤ | 同任务 FAIL 3 轮 | **不停**，标 [~] + cascade skip 下游，进下一里程碑 |
