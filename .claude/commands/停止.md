# /停止

如果你正在跟一个 run 互动（看到主 Agent 在 for-loop 中跑任务），敲 `/停止` 让它**在当前任务结束后**优雅停下，而不是粗暴关窗口。

> **说明**：关窗口也能停，但 active.json status 不会变；下次 /继续 时主 Agent 会以为是异常中止。
> /停止 命令显式写状态便于追溯。

---

## 执行步骤

### Step 1：检查 active.json

```bash
cat .claude/runs/active.json 2>/dev/null
```

不存在 → 汇报「无活跃 run，没有可停止的流水线」。结束。

### Step 2：判断当前是否真在跑

读 `tasks_running.json`：
- 存在且非空 → 说明真在跑某条任务
- 不存在或空 → 说明已经在任务间空档；直接停下即可

### Step 3：写 stopped.flag + 更新 active.json

```bash
RUN_ID=<from active.json>
echo "$(date -u +%Y-%m-%dT%H:%M:%SZ) USER_STOP" >> .claude/runs/$RUN_ID/timeline.log
touch .claude/runs/$RUN_ID/stopped.flag
```

更新 active.json：
```json
{ ..., "status": "stopped", "stopped_at": "<now>" }
```

### Step 4：如果当前任务还在跑

由于主 Agent 是同一进程跑 for-loop，/停止 触发时实际上 for-loop 会立即看到 stopped.flag → 结束当前任务（如果在 dev 阶段，让 dev 把当前编辑保存好；如果在 tester 阶段，让 tester 跑完再停）→ 然后退出 for-loop。

**优雅停止规则**（按当前阶段）：
- 在 developer 阶段 → 让 developer 完成本轮（不要中途留半截代码）→ 不调 tester → 标该任务 status=interrupted（下次 /继续 时进 attempt+1 重新跑）
- 在 tester 阶段 → 让 tester 跑完判 PASS/FAIL → 按正常 PASS commit / FAIL 累积 → 然后停
- 在任务间空档 → 立即停

### Step 5：汇报

```markdown
## /停止 — Run <run-id> 已优雅停止

**停止时**：<timestamp>
**当前进度**：M2 里程碑（用户认证），任务 12/20
**最后任务**：P3.5 — PASS 已 commit (a1b2c3d)
**未完成任务**：P3.6 google_oauth — interrupted at attempt 1（下次 /继续 从 attempt 2 起跑）

下次任意窗口敲 `/继续` 即可接着跑。
```

---

## 硬约束

- ❌ 不要在停止时**回滚**已完成任务的 commit
- ❌ 不要在停止时**清理** run 目录（删 = `/丢弃 <run-id>` 干的活）
- ❌ 不要在停止时**自动 push**（push 由里程碑结束触发；停止时如果有未 push 的 commit，在汇报里提醒「有 K 个本地 commit 未 push」）
- ✅ 必须写 timeline.log + stopped.flag
- ✅ 优雅停 ≠ 立即 kill；当前任务跑完才能停（避免半截代码状态）
