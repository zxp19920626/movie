## A2 cp 自升级 SDK（HMAC 签名 + 弹窗策略）

参见 [`../channel-module-design.md`](../channel-module-design.md) 第 4 章。

- [ ] **A2.1** `CpClient`：
  - HTTP 客户端（OkHttp）+ baseUrl 从 BuildConfig.CP_BASE_URL 读
  - 拦截器自动加 `X-CP-Timestamp` + `X-CP-Signature`（HMAC-SHA256 over `app_id\nversion_code\nchannel\ndevice_id\ncountry\ntimestamp`）
  - 重试策略：401（HMAC 失败）/ 429（限流）不重试；5xx 指数退避 3 次
- [ ] **A2.2** `SelfUpgradeManager.checkUpgrade(context)`：
  - 仅 `BuildConfig.SELF_UPGRADE_ENABLED && BuildConfig.CHANNEL != "gp"` 时调（红线 #2 第二道闸）
  - 调 `/api/v1/cp/upgrade/check` 拿规则
  - 返回 `UpgradeResult` 数据类（has_update / target / is_force / can_skip / popup_strategy / 文案 i18n / download_url / sha256 / size / changelog）
- [ ] **A2.3** 弹窗策略（6 种）持久化（DataStore）：
  - `once_per_launch` / `once_per_session` / `once_per_day` / `once_per_week` / `once_per_release` / `custom_interval`（hours）
  - 每次 popup 记 `(target_version_code, last_shown_at)` 决定下次是否再弹
- [ ] **A2.4** `UpgradeDialog`（Compose）：标题 / 内容 / 跳过 / 立即更新；多语言文案从 `popup_*_i18n` 按 system locale 选
- [ ] **A2.5** `UpgradeDownloader`：DownloadManager 下 `download_url` → 校验 SHA256 → 触发 ACTION_VIEW APK install intent
- [ ] **A2.6** 强升流程：`is_force=true` 时拦截 Activity 直到下载完成；不允许 back 退出
- [ ] **A2.7** Application 启动时调 SelfUpgradeManager.checkUpgrade（异步 + 失败静默）
- [ ] **A2.8** `device_id` 生成策略：随机 UUID 写入 DataStore（**不要用 ANDROID_ID**，多账号同一设备会变，且 Android 8+ 各 App 不一致）
