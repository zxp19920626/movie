## 2. 两条产品线

Movie 仓库实际上是**两个产品**共存于一个 monorepo：

### 2.1 海外影视 App（主业务）
向海外用户提供影视点播服务；C 端 App + 内容运营后台。

### 2.2 App 分发平台（多租户 SaaS）
管理多个 App 的 Android 多渠道包 + 自升级（强制 / 灰度 / 分组下发）。

**关键定位**：这不是 movie 的内部模块，而是**独立的内部 SaaS**：
- 模块代码独立（`backend/app/modules/channel_pack/`）
- 数据多租户隔离（所有 `cp_*` 表带 `app_id` FK）
- 对外 API 中性（`/api/v1/cp/...`，不绑 movie）
- movie 是它的第一个租户；未来可服务其它影视/工具类 App
- 物理上 v1 与 movie 同 docker-compose；DAU > 10w 且接入第 2 个 App 后可抽离独立部署

**为什么独立**：用户长期会管理多个 App，渠道签名 + 自升级是高复用模块；从 day 1 隔离比起来日重构成本低得多。

## 3. 用户角色

| 角色 | 端 | 主要场景 |
|---|---|---|
| **C 端用户** | App | 看片：浏览首页 / 搜索 / 播放 / 看历史 / 登录注册 |
| **内容运营** | admin-web | 影片上下架 / 多语言文案 / 地区可见性矩阵 / 二次合规审核 / 推荐位 |
| **技术运维** | 阿里云控制台（VOD + RDS + ECS + OSS）+ admin-web | 上传母片 / 跑转码 / 改 RAM / 看监控 / 故障应急 |
| **App 分发平台租户管理员** | admin-web | 管理自家 App：渠道 / 版本 / 升级规则 / 签名任务 |
| **超级管理员** | admin-web | 创建租户 App / 分配 admin / 配 RBAC / 看全局仪表盘 |

**RBAC 权限矩阵**：6 大模块（dashboard / cp / content / user / membership / permissions）× view/edit；4 个 seed 角色（Super_Admin 内建全权 / Content_Manager / Global_Ops / Service_Auditor 只读）。`a_admin_users.app_scope` 控制每个 admin 看哪些 App 的数据（多租户隔离）。

## 4. 功能模块清单

### 4.1 海外影视 App（content + user 模块）

#### 4.1.1 用户认证
- Email 注册 / 登录（密码 bcrypt + 强度校验）
- Google OAuth 登录（Web + Android Client ID 各一份）
- 手机 OTP 登录（前置 Cloudflare Turnstile + 国家号段白名单 + 三层频控：号 60s / IP 5min/5 次 / 设备 10 次/天）
- WhatsApp OTP（高费率国家替代短信，未实现）
- JWT access 15min + refresh 30d + 旋转 + 旧 refresh 黑名单
- 设备指纹注册（user + device + app 唯一）
- 用户主动删除账号入口（`POST /users/me/delete-request`，软删 + 撤销所有 RefreshToken；30 天后异步真删 PII）

#### 4.1.2 影视内容
- 影片基本字段：code（业务编码 unique）/ 标题 / 描述 / 类型（movie/series/variety/anime）/ 分类 / 标签 / 评分 / 年代 / 时长 / 导演 / 演员 / 制片公司 / 封面 / 海报 / 预告片
- 多语言：title_i18n / description_i18n / changelog_i18n（9 种语言占位）
- 剧集（series 类型用，按 season + episode_no）
- VOD 关联：vod_file_id + 同步快照（vod_status / vod_duration / vod_cover_url / vod_synced_at）
- 状态机：draft / published / offline + 二次审核 draft / pending / approved / rejected
- 推荐位：featured / trending / recommend_priority / views
- **地区可见性矩阵**（`ct_region_visibility`）：影片 × 国家二维矩阵；**默认 false**，运营逐国手动 enable
- 付费分级字段（required_tier）保留为 free 占位；v1 不启用

#### 4.1.3 浏览 / 搜索 / 播放
- 列表：按 user.country 过滤地区可见性 + 状态/二审/VOD 三重门 + 分类/类型筛选
- 详情：不可见返 404 不返 403（避免存在性泄露）
- **首页聚合**：featured + continueWatching + trending + top10 单接口减少 App 端往返
- **搜索**：按 code/director/studio/cast/tags/title_i18n/description_i18n 多字段（v1 ilike + JSON 文本兜底；DAU 起来后换 MySQL 全文索引或 ES）
- **播放 token**：`GET /videos/{id}/play-token` 5min TTL + IP 绑定 + 走 IPlayTokenProvider 抽象
- **观看历史**：upsert (user_id, video_id, episode_id) 自动 prune 保留每用户最近 200 条

#### 4.1.4 防盗链最小三件套
- HLS 加密 + 短期签名 URL（5–10min）+ UA / 包签名校验
- App Sign 中间件：X-App-Ts + X-App-Sig (HMAC-SHA256(secret, "ts|UA")) ±5min skew，整组 `/videos/*` 强制
- Redis 单 IP 视频并发限流
- **不**自建 m3u8 代理（VOD 已做完）；**不**上 DRM Widevine（贵）

### 4.2 App 分发平台（channel_pack 模块）

#### 4.2.1 租户管理
- 创建 App（仅超管）→ 自动生成 `tenant_uuid`（公开 ID）+ `api_key`（服务端调用，hash 存 DB）+ `hmac_secret`（App SDK 签名用，明文一次性返回）
- 重生密钥（密钥泄露应对）：旧 secret 进 Redis 黑名单 24h 过渡
- 一个 admin 通过 `app_scope` 字段决定能看哪些 App 数据

#### 4.2.2 渠道管理
- code（同 App 内唯一，如 gp / apkpure / direct / huawei）+ name + is_play_store + signing_strategy（walle / none / play_signed）+ enabled + priority + oss_prefix
- **Play 渠道隔离三道闸**：编译期 productFlavors 不打包 SelfUpgradeManager + 运行时 BuildConfig.CHANNEL == "gp" 不调 + 后端 is_play_store=true 直接返 has_update:false

#### 4.2.3 版本管理
- 母包上传：multipart 直传（v1）/ OSS STS 直传（生产，避开 ECS 出口带宽）
- **version_code 严格递增**：重复或回退直接 400（防灰度回滚踩坑）
- changelog_i18n 多语言文案
- 状态机：draft → signing → ready → archived
- finalize：触发 fan-out 创建签名 jobs

#### 4.2.4 渠道签名（Walle 工作流）
- finalize 时为每个 enabled & 非 is_play_store 的渠道创建 cp_apk_signing_jobs
- **幂等**：idempotency_key = hash(app_id + version_code + channel + master_sha256)，同 key 已 success 直接复用
- 防重复消费：SELECT FOR UPDATE SKIP LOCKED
- 步骤：OSS 下母包 → walle CLI 注入 → SHA256 → 上传 OSS → CDN 预热 → 写回
- 失败 retry 3 次（autoretry_for + backoff + jitter）；全失败 status=failed admin 手动 retry
- 全部 jobs success → 自动置 cp_app_versions.status=ready
- **必须预生成不能实时**：1M DAU 实时签名 = ECS CPU 炸 + CDN 命中率 0 + 出口带宽爆

#### 4.2.5 升级规则引擎
- 命中条件：app_id + 版本范围 + 渠道 + 国家 + 灰度区间（device_id_hash_mod 0–99）+ 生效窗
- 命中策略：target_version_code + is_force + can_skip + popup_strategy（once_per_launch / once_per_session / once_per_day / once_per_week / once_per_release / custom_interval）+ 4 个 i18n 文案（title / content / confirm / cancel）+ priority
- 同一 target 上挂多条规则表达「强升 + 灰度」组合
- Redis 缓存 key `cp:upgrade:{app_id}:{version_code}:{channel}:{country}:{hash_bucket}` TTL 5min
- rule 改动 invalidate `cp:upgrade:{app_id}:*`
- **预览功能**：输入 sample 设备返回命中规则 + debug_steps（运营调试用）
- 性能目标 p50 < 50ms（Redis 命中）/ p95 < 200ms（DB）/ p99 < 500ms

#### 4.2.6 公开 API（HMAC 保护）
- `GET /api/v1/cp/upgrade/check?app_id=&version_code=&channel=&device_id=&country=`
  - 头 `X-CP-Timestamp`（5min 内有效防 replay）+ `X-CP-Signature`（HMAC-SHA256 常数时间比较）
  - Cloudflare WAF 限流 5 req/min/device + 后端二次校验
- `GET /api/v1/cp/apk/{tenant_uuid}/{channel_code}/{version_code}` → 302 跳 CDN 签名 URL
- `GET /api/v1/cp/healthz`

### 4.3 VOD 与内容运营（A1' 方案）

| 谁 | 登哪 | 干什么 |
|---|---|---|
| 技术运维 | 阿里云 RAM 子账号 → VOD 控制台 | 上传母片、转码模板（仅 480p+720p）、媒资批量管理 |
| 内容运营 | 自家 admin-web | 地区可见性矩阵 + 二次合规审核 + 上下架 + 推荐位 + 多语言标题 |

**双轨同步**：
- 推送轨：VOD MessageCallback → `/internal/vod/webhook`（HMAC 验签）→ 更新本地 ct_videos 同步快照字段
- 拉取轨：每天 03:00 celery-beat 跑 `sync_vod_metadata` 全量对账 + admin 手动 "Sync All" 按钮
- 每日 reconcile job：调 VOD ListMedia 全量 diff，输出"本地有 VOD 没"和"VOD 有本地没"两份告警
- 不一致兜底：vod_status != ready 的影片，App 端列表自动过滤避免 404

**地区合规**：中东禁酒 / 印尼涉赌 / 马来 LGBT / 巴西禁毒等高敏感剧情，未来接 ChatGPT/Gemini 摘要预审 + 标签（v1 人工二审）。

### 4.4 埋点 / 分析 / 仪表盘

- 事件表 `s_analytics_events`：16 种事件类型白名单（page_view / play_start / play_complete / search / upgrade_check / ad_impression / ad_click 等）；4 个 composite index（type+time / user+time / app+time / page+time）；行级追加只插不更
- `POST /api/v1/analytics/track`：公开 + optionalAuth + 100/min/IP 限流 + 单条/批量（最多 50）
- 仪表盘：DAU/DAA + play_start/search/upgrade_check/ad_pv + 30 天趋势 + 留存 + 广告位分布（ECharts）
- 周期切换：24h / 7d / 30d；trend 切换：7/14/30/90d

### 4.5 管理后台（admin-web）

模块化目录与 backend 对齐（`channel-pack` / `content` / `user` / `admin`）。

主要页面：
- 登录 + Layout（按 hasPermission 过滤菜单 + 显示当前租户）
- App 分发平台：App 租户管理（仅超管 + 一次性密钥显示对话框 + 重生密钥）/ 渠道 / 版本 / 升级规则（含预览）/ 签名 job（含 retry）
- 内容：分类（多语言 9 种 + 父子 + 软删）/ 影片（4 tab 编辑 + 地区可见性矩阵 + 二审 + VOD 同步按钮 + 推荐位）
- 用户管理：列表 / 搜索（共 3 端点）
- 权限：角色矩阵勾选 + 管理员 CRUD + 角色分配 + app_scope
- 仪表盘：6 KPI 卡 + 4 ECharts line 图 + period/trend 切换

## 7. 关键风险（红线摘要）

完整红线见 [`架构与红线.md`](架构与红线.md) 第 5 章。这里只列**最高优**：

1. **keystore 不许进 Git** — 三地备份（1Password + 离线 GPG U 盘 + Play App Signing 托管）
2. **Play 包硬关自升级** — 编译期 flavor 隔离 + 运行时不调 + 后端硬拒，不靠运行时判断
3. **视频不许明文 m3u8** — HLS 加密 + 短期签名 + UA/包签名校验 + 并发限流
4. **CDN/SMS/VOD 必须设月度上限** — 30/60/90% 告警 + 预付费；禁后付费裸奔
5. **手机 OTP 必须前置 Turnstile** + 国家号段白名单 + 三层频控
6. **VOD ↔ 本地 reconcile 每日全量 job** 必须有
7. **/upgrade/check 必须 HMAC 签名** — 防伪造 device_id 刷量
8. **ct_region_visibility 默认 false** — 运营手动开启每个国家可见性
